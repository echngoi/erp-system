════════════════════════════════════════════════════
  HƯỚNG DẪN DEPLOY FRONTEND LÊN VERCEL
  Domain Backend: https://hrmgo.site
════════════════════════════════════════════════════


────────────────────────────────────────────────────
BƯỚC 1: ĐĂNG KÝ VERCEL
────────────────────────────────────────────────────

1. Mở trình duyệt → vào https://vercel.com
2. Click "Sign Up" → chọn "Continue with GitHub"
3. Đăng nhập bằng tài khoản GitHub (echngoi)
4. Cho phép Vercel truy cập repositories


────────────────────────────────────────────────────
BƯỚC 2: IMPORT PROJECT
────────────────────────────────────────────────────

1. Sau khi đăng nhập → click "Add New..." → "Project"
2. Tìm repository "erp-system" → click "Import"
3. Nếu không thấy repo:
   - Click "Adjust GitHub App Permissions"
   - Chọn repo "erp-system" → Save
   - Quay lại Vercel → Import


────────────────────────────────────────────────────
BƯỚC 3: CẤU HÌNH PROJECT
────────────────────────────────────────────────────

Ở trang "Configure Project", điền chính xác như sau:

[3.1] Project Name
  → Nhập: erp-frontend (hoặc tên bạn muốn)

[3.2] Framework Preset
  → Chọn: Vite

[3.3] Root Directory
  → Click "Edit" → nhập: frontend → click ngoài để xác nhận
  ⚠️  QUAN TRỌNG: PHẢI điền "frontend" vì code FE nằm trong thư mục frontend/
  Nếu không điền → Vercel sẽ build sai

[3.4] Build & Output Settings (để mặc định, không cần sửa)
  - Build Command: npm run build  (tự nhận từ vercel.json)
  - Output Directory: dist         (tự nhận từ vercel.json)
  - Install Command: npm install   (tự nhận từ vercel.json)


────────────────────────────────────────────────────
BƯỚC 4: THÊM BIẾN MÔI TRƯỜNG (Environment Variables)
────────────────────────────────────────────────────

⚠️  ĐÂY LÀ BƯỚC QUAN TRỌNG NHẤT — kết nối FE với BE

Ở mục "Environment Variables" trên cùng trang Configure Project:

[4.1] Biến thứ 1:
  Key:   VITE_API_BASE_URL
  Value: https://hrmgo.site/api
  → Click "Add"

[4.2] Biến thứ 2:
  Key:   VITE_WS_URL
  Value: wss://hrmgo.site
  → Click "Add"

[4.3] Environments: đảm bảo tích chọn cả 3:
  ☑ Production
  ☑ Preview
  ☑ Development

Giải thích:
  - VITE_API_BASE_URL: FE gọi API backend qua URL này
    → Code trong frontend/src/services/api.js:
       const BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:8000/api';
  - VITE_WS_URL: FE kết nối WebSocket real-time qua URL này
    → Code trong frontend/src/pages/Attendance/LiveMonitor.jsx:
       const WS_URL = (import.meta.env.VITE_WS_URL || 'ws://localhost:8000') + '/ws/attendance/';


────────────────────────────────────────────────────
BƯỚC 5: DEPLOY
────────────────────────────────────────────────────

1. Click nút "Deploy" → đợi 1-3 phút
2. Vercel sẽ hiển thị:
   - Installing dependencies...
   - Building...
   - Deploying...
3. Khi thành công → thấy "Congratulations!" với URL

URL sẽ có dạng:
  https://erp-frontend-xxxx.vercel.app

⚠️  GHI LẠI URL NÀY — cần dùng ở Bước 7


────────────────────────────────────────────────────
BƯỚC 6: KIỂM TRA FE HOẠT ĐỘNG
────────────────────────────────────────────────────

1. Click vào URL Vercel vừa tạo
2. Phải thấy trang đăng nhập ERP
3. Đăng nhập bằng tài khoản superuser đã tạo trên VPS
4. Nếu đăng nhập thành công → FE đã kết nối với BE ✅

Nếu đăng nhập lỗi "Không kết nối được máy chủ":
  → Xem Bước 7 (cập nhật CORS)


────────────────────────────────────────────────────
BƯỚC 7: CẬP NHẬT CORS TRÊN BACKEND (VPS)
────────────────────────────────────────────────────

⚠️  Backend cần cho phép FE từ domain Vercel gọi API

[7.1] SSH vào VPS
ssh root@123.30.48.72

[7.2] Sửa file .env.production — thêm domain Vercel vào CORS
# THAY erp-frontend-xxxx.vercel.app bằng URL thật từ Bước 5

nano /opt/erp/backend/.env.production

# Tìm dòng CORS_ALLOWED_ORIGINS, sửa thành:
CORS_ALLOWED_ORIGINS=https://hrmgo.site,https://www.hrmgo.site,https://erp-system-nine-zeta.vercel.app

# Lưu file: Ctrl+O → Enter → Ctrl+X

[7.3] Restart backend
cd /opt/erp && docker compose restart backend

[7.4] Đợi backend healthy
sleep 15 && docker compose ps
# Phải thấy erp-backend-1 Up (healthy)

[7.5] Test lại trên trình duyệt
# Mở lại URL Vercel → đăng nhập


────────────────────────────────────────────────────
BƯỚC 8: GÁN CUSTOM DOMAIN CHO VERCEL (TÙY CHỌN)
────────────────────────────────────────────────────

Nếu muốn FE chạy trên domain riêng (VD: app.hrmgo.site):

[8.1] Vào Vercel Dashboard → Project → Settings → Domains
[8.2] Nhập: app.hrmgo.site → click "Add"
[8.3] Vercel sẽ yêu cầu tạo DNS record:
  Loại: CNAME
  Tên: app
  Giá trị: cname.vercel-dns.com
[8.4] Vào trang quản lý DNS domain hrmgo.site → thêm record trên
[8.5] Đợi 5-15 phút → Vercel tự xác nhận + cấp SSL

Sau khi xong:
  - FE: https://app.hrmgo.site (Vercel)
  - BE: https://hrmgo.site (VPS BizflyCloud)

⚠️  Nhớ thêm https://app.hrmgo.site vào CORS_ALLOWED_ORIGINS trên VPS


────────────────────────────────────────────────────
TỰ ĐỘNG DEPLOY KHI PUSH CODE MỚI
────────────────────────────────────────────────────

Mỗi lần bạn push code lên GitHub branch master:
  → Vercel TỰ ĐỘNG build và deploy FE mới
  → Không cần làm gì thêm

Kiểm tra: GitHub → push code → vào Vercel Dashboard → Deployments
  → Thấy build mới đang chạy


────────────────────────────────────────────────────
SƠ ĐỒ KẾT NỐI FE ↔ BE
────────────────────────────────────────────────────

Trình duyệt người dùng
    │
    ├── HTML/CSS/JS ← Vercel (https://erp-frontend-xxxx.vercel.app)
    │
    ├── API calls ──→ VPS Backend (https://hrmgo.site/api/...)
    │                    │
    │                    ├── Django + Daphne (port 8000)
    │                    ├── PostgreSQL (port 5432)
    │                    └── Redis (port 6379)
    │
    └── WebSocket ──→ VPS Backend (wss://hrmgo.site/ws/attendance/)


────────────────────────────────────────────────────
XỬ LÝ LỖI THƯỜNG GẶP
────────────────────────────────────────────────────

[Lỗi 1] Build failed - SyntaxError CSS
  → Kiểm tra file frontend/src/index.css có dấu } thừa không
  → Sửa trên local → push lại

[Lỗi 2] Đăng nhập lỗi "Network Error" / "CORS"
  → Backend chưa cho phép domain Vercel
  → Làm Bước 7 (thêm CORS)

[Lỗi 3] Trang trắng / 404
  → Kiểm tra Root Directory = "frontend" trong Vercel Settings
  → Kiểm tra vercel.json có rewrites đúng

[Lỗi 4] API trả về "Authentication credentials were not provided"
  → Đúng rồi, FE cần gửi token → đăng nhập trước

[Lỗi 5] WebSocket không kết nối
  → Kiểm tra VITE_WS_URL = wss://hrmgo.site (có wss://, không phải ws://)
  → Kiểm tra Nginx có proxy WebSocket đúng không
